# DeepVault Guide

Welcome to DeepVault — an interactive learning fortress for mastering C++, Unreal Engine, and the computer systems beneath them.

## How This Works

Every folder on the dashboard contains **Learn** notes and **Quiz** exercises:

| Mode | What It Does |
|---|---|
| **Learn** | The full vault note, rendered exactly as it appears in Obsidian. Read, scroll the right-side table of contents, click wiki links to jump between notes. |
| **Quiz** | Randomized questions per note — multiple choice and text input. Instant feedback. Scores saved locally on your device. |

### Tracking Your Progress

Progress is based on **quiz scores only** — reading a note does not count. Here's how it works:

| Status | What It Means |
|---|---|
| **NOT STARTED** | You haven't taken the quiz yet |
| **IN PROGRESS** | At least 1 attempt, but best score is below 80% |
| **COMPLETED** | You scored 80% or higher on at least one attempt |

The folder header shows your overall completion percentage. Quiz scores and attempt history are saved locally via localStorage — they never leave your device.

### Bookmarks

**Bookmark notes** you want to return to — click the ☆ star on any note card in the Study tab. Bookmarked notes appear under the ⭐ Bookmarks tab for quick access.

- Click ☆ to bookmark → turns to ⭐ (gold)
- Click ⭐ to remove bookmark
- Switch to the Bookmarks tab to see only your bookmarked notes
- Bookmarks are included in Export/Import — they travel with your progress

### Tips

**Wiki links** like [[04 — CPU Architecture]] connect notes to each other. Click any link to jump to that note's Learn page.

**Recent notes** appear on the dashboard below your folders — quick jump back to what you were studying.

**Search** for any note by title using the search bar at the top of every page.

**Toggle theme** with the 🌙 / ☀️ button for dark or light mode.

## Master Study Order

This vault is designed to be read in order. Each note builds on the ones before it. See the [[Learning Path]] for the full reading order — this is just the big-picture map.

### Layer 1 — Computer Systems (start here)

Without understanding the hardware, C++ is just magic words. Start with the physical reality.

Covering: bits and bytes → assembly → CPU → memory hierarchy → virtual memory → storage → DMA → networking → boot process → GPU → interrupts → system software → concurrency. From transistor to thread pool.

Start with [[01 — Bits, Bytes & Number Systems]] and follow the [[Learning Path — Computer Systems|Computer Systems Learning Path]].

### Layer 2 — C++

Once you understand the hardware, learn the language that runs on it.

Six phases: Primer (absolute beginner) → Syntax & Types → Memory → OOP & Generics → Modern C++ → Engine Bridge. Every note has exercises.

Start with [[01 — Primer/00 — Primer Overview|C++ Primer]] and follow the [[Learning Path — C++|C++ Learning Path]].

### Layer 3 — Game Math

The mathematics that make 3D worlds possible.

Covering: Trigonometry → Matrices → Quaternions → Collision Detection → Physics Simulation → Animation → Navigation → Rendering → Procedural Generation → Audio & Networking → Geometric Algebra → UE Bridge. Framework-agnostic (pure C++ structs, not UE-specific).

Start with [[From Trigonometry to 3D Transformations]] and follow the [[Learning Path — Game Math|Game Math Learning Path]].

### Layer 4 — Unreal Engine Core

How UE turns C++ into a game engine.

Covering: UHT & Macros → Pointer Ecosystem → String Types → Casting → Containers → Templates → Move Semantics.

Start with [[UHT, Macros & Reflection]] and follow the [[Learning Path — Unreal Engine Core|UE Core Learning Path]].

### Layer 5 — UE Networking

Multiplayer from the hardware up.

Covering: Network Roles → RPCs → RepNotifies → PushModel → Iris Replication → Iris Filtering → GAS Networking.

Start with [[Network Roles]] and follow the [[Learning Path — Unreal Engine Networking|UE Networking Learning Path]].

### Layer 6 — Big O & Algorithms

Algorithm analysis for game developers.

Covering: Complexity classes → Amortized analysis → Applied examples → Sorting algorithms → Graph algorithms → Dynamic programming.

Start with [[The Big O Complexity Spectrum]] and follow the [[Learning Path — Algorithms & Complexity|Algorithms Learning Path]].

### Running Locally

Double-click `index.html` to launch. Everything stays on your device — no internet required after first load.

### Moving Between Devices

Your quiz progress lives in your browser's localStorage. To continue studying on another device:

1. Export on your current device — downloads a JSON file
2. Transfer the file (email, AirDrop, cloud)
3. Import on the new device — merges all your scores

Export and Import buttons are at the bottom of the dashboard. No account needed.

### Cloud Sync

Sync your progress across all devices with a free GitHub token. Click **☁️ Sync** at the bottom of the dashboard:

1. Create a **classic token** at [github.com/settings/tokens](https://github.com/settings/tokens) — check ONLY the `gist` scope
2. Paste it into DeepVault and click **Connect**
3. Progress auto-saves to a private GitHub Gist after every quiz and bookmark
4. On another device, paste the same token — all progress auto-loads

Your data stays in YOUR GitHub account — private and invisible to anyone else.
