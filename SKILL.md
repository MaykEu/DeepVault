# DeepVault — Static Web App

Build and maintain the DeepVault web application at `D:\User\Desktop\DeepVault\`. This is a pure vanilla HTML/CSS/JS app — zero dependencies, runs by double-clicking `index.html`, deployed via GitHub Pages.

## Content Quality Standards — Pedagogical Philosophy

DeepVault is a **computing education platform** — programming languages, computer systems, game engines, networking, cybersecurity, algorithms, GPU programming. The same content quality principles apply regardless of which computing domain you're teaching.

### What Makes a DeepVault Note

Lessons, not summaries. Every note must follow these rules:

1. **Two-Way Explanation (MANDATORY):** Simple analogy section + Technical deep-dive section. Both REQUIRED.
2. **No shallow summaries:** Deep coverage — 700-1100 line notes are normal.
3. **Hardware-first / Concrete-first:** Start from physical reality before abstractions.
4. **Prerequisites callout (MANDATORY):** List prerequisite notes with wiki links.
5. **See Also (MANDATORY):** Minimum 3 wiki-linked related notes at the end.
6. **Exercises:** 4-5 per note: conceptual, code/application, analysis, real-world.
7. **Show the trap:** Demonstrate WRONG patterns alongside correct ones.
8. **Analogies → Math → Code:** Physical metaphor first, then notation, then implementation.

### Quiz Quality Rules

- **5-15 questions per note MANDATORY.** 1-2 question quizzes are UNACCEPTABLE. Mix MC and text input.
- Every question MUST have an explanation
- FIELD NAMES MUST BE FULL: `question` NOT `q`, `options` NOT `o`, `correctAnswer` NOT `a`, `explanation` NOT `e`. Abbreviated fields cause "undefined" — this broke 115 quizzes.
- JSON ESCAPING: Do NOT backslash-escape single quotes (`\'`). Use `json.dumps()` for ALL JSON output.
- WRITE to `data/quizzes.json`, then run `python build.py`. Never leave intermediate batch files on disk.
- VERIFY: Run `python quiz-cli.py verify` → 0 errors AND 0 warnings. Run `node test.js` → 20/20.

### What DeepVault Is NOT

- ❌ Reference documentation — belongs in a wiki
- ❌ Cheat sheets or bullet-point summaries
- ❌ Lecture transcripts
- ❌ Beginner tutorials that skip the "why"

### Why This Vault Exists

Most computing education is either too shallow or too jargon-dense. DeepVault bridges both — every concept explained twice, grounded in hardware reality, connected via wiki links. The pedagogy IS the product.

## Architecture

**SCALABLE BUILD SYSTEM.** data.js is generated by `python build.py` from 4 independent JSON source files in `data/`:

- `data/notes.json` — all note content (vault → build.py). **Never edit manually.**
- `data/quizzes.json` — all QUIZ_DATA + QUIZ_NOTES. **Write here safely — can't corrupt notes.**
- `data/folders.json` — folder structure per domain.
- `data/reference.json` — glossary, guide, learning paths.

Build process: `python build.py` validates all quizzes → generates `js/data.js`. **If validation fails, build ABORTS.**

```
data/
├── notes.json      ← Note content (vault → build.py)
├── quizzes.json    ← Quiz questions (safe to edit)
├── folders.json    ← Folder structure
└── reference.json  ← Glossary, Guide, Learning Paths

build.py            ← Combines sources → js/data.js (validates first)
quiz-cli.py         ← Safe quiz management (add, verify, backup)
test.js             ← Smoke tests (syntax, CSS, quiz format)
convert.py          ← Vault → data/*.json converter
js/data.js          ← GENERATED — never edit manually
```

**VAULT FIRST — ALWAYS.** Every content change MUST happen in the vault `.md` file FIRST, then sync to data.js.

## Sub-Agent Non-Stop Pattern

**When dispatching quiz creation work, always include explicit non-stop instructions.** Agents naturally pause between batches. Use: "DO NOT PAUSE or ask to continue. Process ALL notes until complete."

## Quiz Creator Instructions Checklist

When instructing an AI to create quizzes, these MUST be in the prompt:

1. FIELD NAMES: Use FULL names — question, options, correctAnswer, explanation. NEVER abbreviate.
2. JSON ESCAPING: Use json.dumps(). Do NOT backslash-escape single quotes.
3. INSERT AFTER WRITING: Write to `data/quizzes.json`, run `build.py` to regenerate `js/data.js`.
4. NON-STOP: "DO NOT PAUSE or ask to continue."
5. VERIFY: After every batch, run quiz-cli.py verify. Fix errors before next batch.

## Table Rendering — Empty First Column Bug

**Tables with an empty first header cell render with headers shifted left.** Fix: give the first column a label (`| **Property** | **FName** |`). Do NOT use `&nbsp;` — renders as literal text.

## Submit Button & Review Mode — See `references/submit-review-pitfalls.md`

Two edge cases: submit button silently fails (correctAnswer vs correctIndex mismatch) and review mode shows empty state when questions exist (q vs question field mismatch). Both require dual-format support. Full quiz format pitfalls in `references/quiz-format-pitfalls.md`.

## Hash Routing — See `references/hash-routing-pitfalls.md`

Same-hash navigation bug: `router.navigate` to current hash is ignored. Fix: append `?t=timestamp` to quiz routes. Also: NEVER build onclick strings via Python — multi-level escaping nightmare. Use router-based solutions.
